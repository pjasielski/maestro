## Description
Field report (2026-07-19, Piotr): choosing "gitignored" session visibility has no effect when the target project already has a `.gitignore` — the installer's `else` branch prints "Exists: .gitignore (preserved)" and never adds `.sessions/`. Since virtually every real project has a `.gitignore` (language templates), the visibility choice was silently ignored for most installs; sessions ended up as pending/tracked files.

Fix in `install.sh` (.gitignore section):
- No `.gitignore` → unchanged behavior (create; now also ignores legacy `sessions/`)
- Existing `.gitignore` + visibility `gitignored` → append `.sessions/` + `sessions/` block, idempotent (grep guard `^\.?sessions/`)
- Existing `.gitignore` + visibility `committed` but `.sessions/` is ignored in it → loud WARNING (never auto-edits the user's ignore rules)
