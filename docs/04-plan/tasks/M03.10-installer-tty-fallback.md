## Description
Field report (2026-07-18): installing from main showed no setup questions. Two known paths cause this silently: (1) `read … </dev/tty` — when /dev/tty is missing/unreadable (some Git Bash/mintty setups, CI), questions fail or fall through to defaults without explanation; (2) reinstall detection — any pre-existing MAESTRO.md (e.g., left by a previously failed run) switches to reinstall mode, which by design skips questions but only says so in one easily-missed line.

## Acceptance Criteria
- [ ] Guard before questions: if /dev/tty unavailable, print a loud notice ("non-interactive shell — using defaults: committed/all tools/async; use --preconfigured with env vars to customize") instead of failing or silently defaulting
- [ ] Reinstall mode announces clearly WHY questions are skipped and how to force a fresh setup
- [ ] Failed-install artifacts don't fake a reinstall: verify MAESTRO.md check, or write a completion marker
- [ ] Tested: curl|bash in a terminal, curl|bash in non-tty, rerun after interrupted install
